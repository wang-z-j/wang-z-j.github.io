生命周期使用示例

场景 1: 函数返回引用

  • 当函数需要返回某个引用的时候，必须确保某个引用于某个输入参数的生命周期相关联，避免悬空引用
  • 示例：获取字符串中更长的部分

场景 2: 数据结构中的引用

  • 在定义包含引用的结构体时，需要显示的为结构体指定生命周期，以保证结构体的声明周期不超过内部引用的声明周期
  • struct Book<'a> 表示 Book 的生命周期 'a 必须小于等于其字段 title 和 author 的生命周期。
  • 如果不使用生命周期标注，编译器将无法判断结构体的生命周期是否安全。

场景 3: 方法返回内部数据的引用

  • 当某个方法需要返回结构体内部的引用时，必须为结构体定义生命周期标注。
  • Container<'a> 确保 value 的生命周期不超过 Container 实例的生命周期。
  • 方法 get_value 返回的引用生命周期与结构体生命周期 'a 保持一致。

场景 4: 泛型与生命周期结合

  • Pair<'a, T> 表示 Pair 存储了指向 T 的两个引用，它们共享生命周期 'a。
  • 方法 first 和 second 返回的引用生命周期与 Pair 的生命周期一致。
  • 在泛型上下文中，生命周期标注常与 impl 一起使用，特别是当泛型类型需要存储引用时。

生命周期消除

每个引⽤参数都有⾃⼰的⽣命周期参数。

如果只有⼀个输⼊引⽤参数，那么它的⽣命周期会被赋予所有输出引⽤。

如果有多个输⼊⽣命周期参数，但其中⼀个是 &self 或 &mut self ，那么 self的⽣命周期会被赋予所有输出引⽤

返回值不包含引用 无需标注

特殊生命周期标注

  • ‘static ⽣命周期表示整个程序运⾏期间都有效的⽣命周期。这通常⽤于全局变量或字符串字⾯量
